What we understood
  • ddl-auto=update — Hibernate reads your Java entity class and auto-creates or updates the matching MySQL table. You never write CREATE TABLE SQL manually.
  • show-sql=true — Every SQL query Spring runs is printed to the IntelliJ console. Essential for understanding what's happening in the database while learning.
  • Enum with properties pattern — Every enum constant is a full Java object. It can carry extra data (like a displayName) alongside its code name. EN("English") stores "EN" in the DB but getDisplayName() returns "English" for the UI.
  • @Enumerated(EnumType.STRING) — Always use STRING, never ORDINAL. ORDINAL stores a number (0,1,2) — if you reorder enum constants, all existing database data breaks silently.
  • Never store images in the database — Store the image URL. The actual file lives on AWS S3. DB stores a short string, app loads the URL. Fast, cheap, scalable.
  • @ManyToOne vs @ManyToMany — A farmer belongs to one district (@ManyToOne). A farmer grows many crops AND one crop is grown by many farmers (@ManyToMany). Hibernate creates the join table automatically.
  • @JoinTable creates the join table automatically — farmer_crops and farmer_animal_types are never written as Java classes. The @ManyToMany + @JoinTable annotation on Farmer.java is enough.
  • FetchType.LAZY — Don't load related data unless specifically asked. Loading a farmer does not auto-load all their crops and animals. This is critical for performance.
  • Derived data rule — Never store data that can be calculated. getFullName() returns firstName + " " + lastName. No fullName column needed in the database.
  • State and District as entities, not enums — Districts are data, not code. 788 rows across all Indian states. Adding a new state = inserting rows, not redeploying code.

📄 enums/farmer/Language.java — enum with displayName property
package com.flora.api.enums.farmer;

public enum Language {

    EN("English"),   // stored in DB as "EN"
    TA("Tamil"),     // stored in DB as "TA"
    HI("Hindi");    // stored in DB as "HI"

    // Each enum constant carries this extra piece of data
    private final String displayName;

    // Constructor — Java calls this automatically for each constant above
    Language(String displayName) {
        this.displayName = displayName;
    }

    // Getter — call Language.EN.getDisplayName() to get "English"
    public String getDisplayName() {
        return displayName;
    }
}

// Usage examples:
// Language.EN.name()           → "EN"       (used in DB, API)
// Language.EN.getDisplayName() → "English"  (used in UI labels)
// Language.TA.getDisplayName() → "Tamil"
📄 entity/farmer/State.java — first entity, no dependencies
package com.flora.api.entity.farmer;

@Entity               // tells Spring: this class maps to a database table
@Table(name = "states") // sets the exact MySQL table name (plural, snake_case)
@Data                 // Lombok: generates getters, setters, toString, equals, hashCode
@NoArgsConstructor    // Lombok: generates empty constructor — required by JPA
@AllArgsConstructor   // Lombok: generates constructor with all fields
public class State {

    @Id                                          // this field is the primary key
    @GeneratedValue(strategy = GenerationType.IDENTITY) // MySQL auto-increments this
    @Column(name = "state_id")                   // MySQL column name in snake_case
    private Long stateId;

    @Column(name = "state_name", nullable = false, unique = true, length = 100)
    // nullable=false → NOT NULL constraint in MySQL
    // unique=true    → UNIQUE constraint, no two states with same name
    // length=100     → VARCHAR(100) in MySQL
    private String stateName;

    @Column(name = "state_code", nullable = false, unique = true, length = 5)
    // 2-letter official code: TN, KA, KL, MH etc.
    private String stateCode;

    @Column(name = "is_active", nullable = false)
    private Boolean isActive = true;

    @OneToMany(mappedBy = "state", fetch = FetchType.LAZY)
    // One state has many districts
    // mappedBy="state" → the 'state' field in District.java owns this relationship
    // LAZY → don't load all districts when you load a state unless asked
    private List<District> districts;
}
📄 entity/farmer/District.java — ManyToOne relationship with State
@Entity
@Table(name = "districts")
@Data @NoArgsConstructor @AllArgsConstructor
public class District {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "district_id")
    private Long districtId;

    @Column(name = "district_name", nullable = false, length = 100)
    private String districtName;

    @ManyToOne(fetch = FetchType.LAZY)
    // Many districts belong to ONE state
    // This is the "owning side" — District holds the foreign key column
    @JoinColumn(name = "state_id", nullable = false)
    // Creates state_id column in districts table → foreign key to states.state_id
    private State state;

    @Column(name = "is_active", nullable = false)
    private Boolean isActive = true;
}

// Result in MySQL:
// districts table has: district_id, district_name, state_id, is_active
// state_id is a FOREIGN KEY referencing states(state_id)
📄 entity/crop/Crop.java — multilingual crop data
@Entity
@Table(name = "crops")
@Data @NoArgsConstructor @AllArgsConstructor
public class Crop {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "crop_id")
    private Long cropId;

    @Column(name = "crop_name", nullable = false, unique = true, length = 100)
    private String cropName;        // English: "Rice"

    @Column(name = "crop_name_tamil", length = 100)
    private String cropNameTamil;   // Tamil: "அரிசி"

    @Column(name = "crop_name_hindi", length = 100)
    private String cropNameHindi;   // Hindi: "चावल"

    @Column(name = "crop_season", length = 50)
    private String cropSeason;      // Kharif / Rabi / Zaid

    @Column(name = "is_active", nullable = false)
    private Boolean isActive = true;

    // Why three language columns instead of one?
    // When app is in Tamil mode → show cropNameTamil
    // When app is in English mode → show cropName
    // No translation library needed on backend — just return the right column
}

📄 entity/farmer/Farmer.java — the central entity, all relationships shown
@Entity
@Table(name = "farmers")
@Data @NoArgsConstructor @AllArgsConstructor
public class Farmer {

    // ── IDENTITY ─────────────────────────────────────────
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "farmer_id")
    private Long farmerId;
    // Auto-incremented by MySQL: 1, 2, 3...
    // You never set this manually

    // ── AUTHENTICATION ────────────────────────────────────
    @Column(name = "phone_number", unique = true, nullable = false, length = 15)
    private String phoneNumber;
    // unique=true → no two farmers with same phone number
    // This is the login identifier — like a username

    @Column(name = "hashed_password", nullable = false)
    private String hashedPassword;
    // BCrypt hash of the real password
    // Real password is NEVER stored — only the hash
    // BCrypt: "password123" → "$2a$10$xyz..." (irreversible)

    // ── BASIC PROFILE ─────────────────────────────────────
    @Column(name = "first_name", nullable = false, length = 50)
    private String firstName;

    @Column(name = "last_name", nullable = false, length = 50)
    private String lastName;
    // Split into first+last so alerts can say "Good morning, Gowtham"
    // getFullName() helper method combines them when needed

    // ── ONBOARDING Q1 ─────────────────────────────────────
    @Enumerated(EnumType.STRING)
    // Store enum as "EN"/"TA"/"HI" in DB — never as 0/1/2
    @Column(name = "preferred_language", nullable = false, length = 10)
    private Language preferredLanguage = Language.EN;
    // Default is English — changed immediately if farmer picks Tamil

    // ── ONBOARDING Q2 ─────────────────────────────────────
    @ManyToOne(fetch = FetchType.LAZY)
    // MANY farmers belong to ONE district
    @JoinColumn(name = "district_id", nullable = false)
    // Creates district_id column in farmers table
    // Foreign key → districts.district_id
    private District district;
    // Farmer selects from dropdown. Drives weather, ML models, market prices.

    // ── ONBOARDING Q3 ─────────────────────────────────────
    @Enumerated(EnumType.STRING)
    @Column(name = "primary_activity", nullable = false, length = 10)
    private PrimaryActivity primaryActivity = PrimaryActivity.CROP;
    // CROP / ANIMAL / BOTH — sets which tabs appear first on dashboard

    // ── ONBOARDING Q4 ─────────────────────────────────────
    @Column(name = "land_size_acres")
    private Double landSizeAcres;
    // Used in fertilizer calculation: kg/acre × landSizeAcres
    // nullable — farmer can skip if primaryActivity is ANIMAL

    // ── ONBOARDING Q5 ─────────────────────────────────────
    @ManyToMany(fetch = FetchType.LAZY)
    // ONE farmer grows MANY crops, ONE crop grown by MANY farmers
    @JoinTable(
        name = "farmer_crops",               // join table name in MySQL
        joinColumns = @JoinColumn(name = "farmer_id"),
        inverseJoinColumns = @JoinColumn(name = "crop_id")
    )
    // Hibernate creates farmer_crops table automatically
    // farmer_crops has exactly 2 columns: farmer_id, crop_id
    private List<Crop> currentlyGrowingCrops;
    // Multi-select dropdown on mobile. No typing.

    // ── ONBOARDING Q6 ─────────────────────────────────────
    @ManyToMany(fetch = FetchType.LAZY)
    @JoinTable(
        name = "farmer_animal_types",
        joinColumns = @JoinColumn(name = "farmer_id"),
        inverseJoinColumns = @JoinColumn(name = "animal_type_id")
    )
    private List<AnimalType> ownedAnimalTypes;
    // Tappable animal icons with Lottie animations during onboarding
    // farmer_animal_types join table auto-created by Hibernate

    // ── ONBOARDING Q7 ─────────────────────────────────────
    @Column(name = "alert_time")
    private LocalTime alertTime;
    // e.g. 06:00 — @Scheduled AlertScheduler reads this per farmer
    // 5AM farmer gets alerts at 5AM. 6PM farmer at 6PM.

    // ── APP STATE ─────────────────────────────────────────
    @Column(name = "is_onboarding_complete", nullable = false)
    private Boolean isOnboardingComplete = false;
    // false = app shows onboarding wizard
    // true  = app shows personalised dashboard

    // ── TIMESTAMPS ────────────────────────────────────────
    @Column(name = "created_at", nullable = false, updatable = false)
    private LocalDateTime createdAt = LocalDateTime.now();
    // updatable=false → Hibernate never changes this after first insert

    @Column(name = "updated_at")
    private LocalDateTime updatedAt = LocalDateTime.now();
    // Updated manually in service layer when farmer profile changes

    // ── HELPER ────────────────────────────────────────────
    public String getFullName() {
        return firstName + " " + lastName;
    }
    // Derived data — never stored in DB. Always computed from first+last.
    // Usage: farmer.getFullName() → "Gowtham Selvaraj"
}

Annotation quick reference
AnnotationWhat it does
@EntityMarks this class as a database table. Hibernate reads it and creates the table in MySQL.
@Table(name = "...")Sets the exact MySQL table name. Without this, Hibernate uses the class name (lowercase).
@IdMarks this field as the primary key of the table.
@GeneratedValue(IDENTITY)MySQL auto-increments this field. You never set the ID yourself.
@Column(name = "...")Sets the MySQL column name. Java uses camelCase, MySQL uses snake_case — this bridges them.
@Column(nullable = false)Adds NOT NULL constraint in MySQL. Spring rejects requests missing this field.
@Column(unique = true)Adds UNIQUE constraint. No two rows can have the same value in this column.
@Column(updatable = false)Hibernate never updates this column after the first insert. Used for created_at.
@Enumerated(STRING)Store enum as its name ("EN") not its position number (0). Always use STRING.
@ManyToOneMany rows of this table belong to one row of another table. Creates a foreign key column.
@OneToMany(mappedBy)One row of this table has many rows in another. No foreign key here — other table holds it.
@ManyToMany + @JoinTableMany-to-many relationship. Hibernate auto-creates a join table with two foreign key columns.
@JoinColumn(name = "...")Sets the foreign key column name in MySQL.
FetchType.LAZYDon't load related data unless explicitly requested. Prevents performance issues.
@Data (Lombok)Auto-generates getters, setters, toString, equals, hashCode. Saves ~200 lines of boilerplate.
@NoArgsConstructorGenerates empty constructor. JPA requires this to create objects and fill fields one by one.
@AllArgsConstructorGenerates constructor with every field as a parameter.
