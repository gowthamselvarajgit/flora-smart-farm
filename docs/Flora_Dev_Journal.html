Day 1 ✓
Spring Boot project setup + MySQL connection
June 2026 · Phase 1 · Foundation
What we built
  • Spring Boot project generated via start.spring.io with Group com.flora, Artifact api, package com.flora.api
  • 6 dependencies added — Spring Web, Spring Security, Spring Data JPA, MySQL Driver, Lombok, Validation
  • flora_db database created in MySQL — ready for Spring to create tables inside it
  • application.properties configured — database URL, credentials, JPA settings, server port 8084
  • Application started successfully — "Started ApiApplication" confirmed in IntelliJ console
# application.properties — Day 1 config
spring.application.name=api
spring.datasource.url=jdbc:mysql://localhost:3306/flora_db
spring.datasource.username=root
spring.datasource.password=root
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQL8Dialect
server.port=8084
What we understood
  • spring.jpa.hibernate.ddl-auto=update means Hibernate reads your Java entity classes and automatically creates or updates the matching MySQL tables. You never write CREATE TABLE SQL manually.
  • spring.jpa.show-sql=true prints every SQL query Spring runs into the IntelliJ console — essential for understanding what's actually happening in the database as you learn.
  • The "Whitelabel Error Page" at localhost:8084 in the browser means Spring Boot is running correctly — it just has no endpoints yet. That's expected.
  • Spring Security is included but not configured yet — it will lock down all endpoints by default until we configure it properly on Day 4.
What comes next — Day 2
  • Create the package structure inside com.flora.api — the 6 folders that organise every class in the project: controller, service, repository, entity, dto, config
  • Understand what each folder does and why separating them matters — this is the architecture pattern every Spring Boot project uses professionally
